\section{Time-Scaled Network Logic}

\subsection{Interfaz de Entrada de Datos Streamlit}

\textbf{El programa transforma una tabla relacional en un grafo.}

\begin{itemize}
    \item Interfaz: Utiliza la librería Streamlit $(st.data_editor)$ para capturar tres variables esenciales por cada tarea: Nombre, Duración $(t)$ y Predecesores.
    \item Estructura: Los datos se almacenan dinámicamente en un DataFrame de Pandas. 
    En esta fase se realiza una limpieza básica (asegurar que t sea numérico y separar los predecesores por comas).
\end{itemize}

\subsection{Motor Lógico del CPM}
\begin{itemize}
    \item Utiliza la librería NetworkX para resolver el modelo matemático a través de Teoría de Grafos.
    \item Validación de Grafo: Convierte las tareas en nodos y las dependencias en aristas dirigidas. 
    primero, valida que sea un grafo dirigido acíclico DAG.Si detecta un bucle, el programa se detiene para evitar ciclos infinitos.
    \item Ordenamiento Topológico: Ordena las tareas de tal forma que ninguna tarea se evalúe antes que sus predecesoras.
    \item Pase hacia Adelante: Calcula el Inicio Temprano (ES) y el Fin Temprano (EF). La regla es: EF = ES + t. El ES de una tarea es el máximo EF de todos sus predecesores.
    \item Pase hacia Atrás: Recorre el grafo en reversa desde la duración total del proyecto para calcular el Inicio Tardío (LS) y el Fin Tardío (LF).
    \item Ruta Crítica: Calcula la Holgura (LS - ES). Si la holgura es exactamente 0, etiqueta la tarea como "Crítica" (verdadero/falso).
\end{itemize}

\subsection{Algoritmo de "Carriles"-Distribución Espacial Y}

Para pasar de datos abstractos a un dibujo donde las tareas no se encimen, el programa ejecuta un algoritmo de asignación de espacios.

\begin{itemize}
    \item Ordenamiento temporal: Ordena las tareas de menor a mayor ES.
    \item Asignación de Coordenada Y: El programa revisa un "carril" (nivel Y). Si el carril está vacío en el tiempo ES de la tarea actual, la coloca ahí. Si ya hay una tarea ocupando ese tiempo, baja al siguiente carril, y así sucesivamente.
    \item Objetivo: Esto garantiza que las líneas de las tareas que ocurren en paralelo nunca colisionen visualmente. A cada tarea se le asigna un valor Y (ej. Y=1, Y=2...).
\end{itemize}

\subsection{Renderizado Cartesiano - Visualización Plotly}

\begin{itemize}
    \item Las Actividades (Líneas Sólidas):
    \item Traza una línea recta entre ambos puntos. Por pura geometría, la longitud de esta línea es exactamente t.
    Aplica formato condicional: Rojo si la tarea es Crítica (Holgura = 0), Azul si no lo es.
    \item Las Dependencias (Líneas Punteadas Grises):
    \item Para conectar una tarea terminada con su sucesora respetando la regla de ortogonalidad (sin diagonales)
    1. Sale del final del predecesor: (EF del predecesor, Y del predecesor).
    2. Baja o sube verticalmente hasta el carril de la nueva tarea: (EF del predecesor, Y del sucesor).
    3. Avanza horizontalmente hasta donde inicia la nueva tarea: (ES del sucesor, Y del sucesor).
    \item La Holgura Libre (Líneas Punteadas de Color):
    \item Si una tarea no es crítica (Holgura > 0), 
    traza una línea punteada desde su fin temprano (EF, Y) hasta su límite máximo permitido (LF, Y). 
    Esto ilustra visualmente el tiempo que tiene esa tarea antes de retrasar todo el proyecto.
\end{itemize}
